# **🧠 大模型端云协同智能路由网关 (Edge-Cloud RL Router)**

基于 **多维语义特征 (BGE-small)** 与 **PPO 强化学习** 的大语言模型动态自适应算力分发系统。

本系统旨在解决千亿级大模型在真实工业落地中面临的\*\*“端侧显存受限、弱网高时延、云端高并发及 API 成本高昂”\*\*等核心痛点。通过提取 Prompt 的多维语义复杂度特征，并结合实时物理环境（网络拥塞、端侧负载、云端队列），利用强化学习智能体（RL Agent）在四种异构算力调度策略中寻找“时延-精度-成本”的帕累托最优解。

## ---

**🌟 核心技术亮点**

1. **零人工标注的认知难度评估**：基于大模型自我一致性（Self-Consistency）与特征向量空间距离（Cosine Similarity）自动量化 Prompt 的认知难度。  
2. **防特征坍缩的多任务网络 (MTL)**：采用多任务学习架构，将高维文本映射压缩至 32 维富语义稠密向量，避免单一标量难度导致的信息丢失。  
3. **消除 Sim2Real 鸿沟的 RL 训练**：摒弃纯随机噪声环境，构建由真实业务数据集驱动的 Gymnasium 物理沙盒，解决强化学习仿真到现实的迁移难题。  
4. **防范分布外泛化 (OOD) 降智的工业护栏**：在 RL 决策层外包裹硬性安全策略（Guardrails），完美阻断极端状态下的 Reward Hacking（奖励作弊）与降智决策。

## ---

**🛠️ 核心技术路线详解 (Technical Route)**

本项目的技术路线由“感知层（评估器）”与“决策层（路由器）”两部分深度解耦并串联而成，全链路流程如下：

### **Phase 1: 多维任务特征感知与难度量化 (Perception Layer)**

大语言模型路由的核心前提是：**系统必须在不运行大模型的情况下，提前“猜出”问题的难度。**

1. **自动标注流水线 (auto\_labeling\_pipeline.py)**  
   * **原理**：利用 LLM 针对同一 Prompt 生成多次回答，利用 BGE-small 提取回答的特征向量并计算平均余弦相似度。  
   * **核心逻辑**：相似度越高，说明模型内部认知越确定（难度低，熵极小）；差异越大，说明模型正在产生幻觉（难度高，熵极大）。系统据此自动生成带有认知难度（Entropy）、序列长度和领域类别的训练集。  
2. **多任务特征提取网络 (MultiTaskDifficultyEvaluator)**  
   * **挑战**：如果直接把大段文本输入给 RL，维度爆炸且 RL 难以收敛；如果只输入一个 0\~1 的难度标量，会丢失领域特征（如代码题与闲聊题的决策逻辑完全不同）。  
   * **解决方案**：引入多任务学习（MTL），构建一个共享底层特征的神经网络。网络在预测标量难度、长度、类别的同时，倒逼中间层提取出一个极具表现力的 **32 维稠密特征向量 ($v\_d$)**。这个 32 维向量就是 RL 路由器洞察任务本质的“眼睛”。

### **Phase 2: 基于强化学习的动态调度博弈 (Decision Layer)**

1. **定义物理环境沙盒 (EdgeCloudRoutingEnv.py)**  
   * 基于 Gymnasium 构建定制化环境。  
   * **状态空间 (State Space)**: 35 维连续向量 \= \[32 维任务特征\] \+ \[网络拥塞度, 手机负载, 云端队列\]。  
   * **动作空间 (Action Space)**: 4 种离散算力策略。  
     * 0: 📱 本地直出 (Edge Only) \- 极低难度，端侧消化。  
     * 1: ☁️ 云端全量 (Cloud Only) \- 极高难度，防范幻觉。  
     * 2: 🧩 任务分解 (Decompose) \- 长文本弱网协同。  
     * 3: 🤝 端云验证 (Edge+Cloud Verify) \- 兜底与防翻车策略。  
   * **奖励塑形 (Reward Shaping)**: 采用复杂的非线性奖励惩罚机制（包含指数级网络惩罚），精确平衡 w\_acc(精度)、w\_lat(时延) 和 w\_cost(成本)。  
2. **彻底解决 Sim2Real 鸿沟 (extract\_real\_features.py)**  
   * **痛点**：RL 在随机生成的假数据下训练极易产生过拟合，上线真实数据瞬间崩溃。  
   * **方案**：使用预训练的评估器将 1000+ 真实样本前向传播，提取出真实的 32 维向量库（real\_vd\_features.npy）。Gymnasium 环境每一步 reset() 均从该真实特征库中采样，让 PPO 智能体在仿真阶段就经受真实数据纹理的毒打。

### **Phase 3: 工业级网关与 Web 可视化部署 (Deployment Layer)**

1. **端到端推理闭环 (main.py & app.py)**  
   * 采用 Streamlit 搭建极具科技感的交互式控制台，支持人类动态拖拽滑动条模拟“弱网”、“手机卡顿”等极端物理状态，并实时触发路由决策。  
2. **OOD 安全护栏 (Guardrails Mitigation)**  
   * **痛点**：面对“千亿高并发架构”等完全超出训练分布（OOD）的变态级提问，PPO 可能会为了规避网络惩罚而错误地选择“本地直出（贪婪策略）”。  
   * **方案**：在网关出口层强行引入硬约束拦截代码。一旦 UI 左侧雷达探测到的真实归一化难度 norm\_difficulty \> 0.65，而 RL 输出动作 0 时，系统将立即触发拦截报警，强制将其重定向至端云协同计算，确保工业级 99.99% 的系统可靠性。

**📂 项目结构 (Project Structure)**

Plaintext

├── auto\_labeling\_pipeline.py    \# 阶段一：自动构建基于语义一致性的高维度难度数据集  
├── multitask\_evaluator\_finetuned.pth \# 阶段一：微调好的多任务特征提取器权重 (灵魂)  
├── extract\_real\_features.py     \# 阶段二准备：提取真实推理环境下的 32 维向量库，解决 Sim2Real 鸿沟

├── EdgeCloudRoutingEnv.py       \# 阶段二：定制化 Gymnasium 端云路由物理沙盒与奖励引擎  
├── ppo\_dynamic\_router.zip       \# 阶段二：PPO 强化学习大模型路由器权重  
├── main.py                      \# 终端验证脚本：端到端连通评估器与路由器  
└── app.py                       \# 阶段三：Streamlit 可视化控制台与 OOD 安全护栏部署
